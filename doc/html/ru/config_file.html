Конфигурационный файл¶

  1. Конфигурационный файл может определять все не заданные в командной строке параметры, кроме -c
  2. Параметры в конфигурационном файле записываются по длинному наименованию параметра командной строки без ведущих дефисов
  3. Причем параметры какой-либо секции (шаблона) перезаписывают параметры определенные без секции. Данное правило реализуется конкретным действием в зависимости от запрограммированных сценариев и может отличаться в версиях.
  4. Параметр action может быть задан в конфигурационном файле только глобально
  5. Параметр params предназначен для сбора всех неименованных параметров командной строки. Каждое действие программы обрабатывает эти параметры по запрограммированному смыслу. Например, для отправки смс, набор данных в params будет обрабатываться как номера телефонов.

token      = 587d68b3aa1d80064a5b923f348b33fe688d8437
action     = balance

verbose    = 1

smsurl    = https://xml.sms16.ru/xml/index.php
stateurl  = https://xml.sms16.ru/xml/state.php

[sms1]
originator = Test2
text   = Отправим парочку тестовых сообщений
params = 79046630770
params = 79046640077
